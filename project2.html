Service Studio Redesign.

Project overview.

As a part of capstone project for the MHCI program at Carnegie Mellon University, we worked with Outsystems to design an intuitive, efficient visual application development platform. Our design aims to create a development environment that allows people without programming skills – Citizen Developers – to be able to rapidly and easily implement an idea. You can check the project website at http://frontier.m-iti.org.

Citizen Developers.

Citizen Developers are the core target of this project, so to guide us in who we are designing for we created some personas. Luís is a senior finance manager and he has been doing this for a long time. In his role, he wants to manage his company expense and has an idea for an application. However, if he goes through the regular process and asks the IT deparment of his company to implement his idea, it can take a long time, possibly more than 6 months. So he wants to be able to it himself really quickly.

Research.

We spent the first half of the project doing research.

Design Process.

Based on what we discovered, we moved into the design phase. Our design process had 3 iterations, with design being followed by prototyping and then user testing. With each iteration, the design was refined based on the findings and feedback from the previous iteration. Keeping in mind the time constraints, we tackled each subsequent iteration with higher fidelity but in a more focused scope.

Paper Prototype.

For the first iteration, we started with a design sprint, created wireframes for the whole system and tested in a paper prototype.

PDF Prototype.

For the second iteration, we created mid-fidelity mocks and tested them in a pdf click-through prototype.

Collaborative Ideation.

Since we gained a lot of valuable feedback from our user tests, we decided to a design sprint similar to the one done on the first iteration, but this time including the client.

Partially working Prototype.

With the ideation results, we created hi-fi mocks for a specific use case and created a partially working prototype in axure.

User tests.

We then tested it out one last time, with actual citizen developers (clients of Outsystems). We also tested with an expert of the current Service Studio to address scalability concerns.

Final Design.

We polished most of our design concepts and validated our design decisions. At this point, we delivered the proejct to Outsystems for further development and implementation.
